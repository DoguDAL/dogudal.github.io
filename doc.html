GitHub Page Documentation

Step 1: Initial Setup

The first step in creating a GitHub page is setting up your repository. Ensure that your repository is public and named username.github.io. This format is necessary for GitHub to map your page automatically at https://username.github.io/. 


            git init
            git remote add origin https://github.com/username/username.github.io.git

Step 3: Publishing the Site

Publishing the site is straightforward, but I initially struggled with branch selection. GitHub Pages requires setting the correct source branch, which in my case was the gh-pages branch: 


            git checkout -b gh-pages
            git push origin gh-pages

I ensured the correct branch was set in the repository’s settings under the GitHub Pages section.

Step 4: Struggles Faced

Managing version control for HTML, CSS, and JavaScript was challenging. Git commands like git log, git stash, and git diff helped me overcome issues with keeping track of different versions.

Step 5: Lessons Learned

Here’s what I learned throughout the process:

  • Effective version control using Git
  • Responsive web design principles
  • Using GitHub Pages to host static websites
  • Troubleshooting deployment issues by checking GitHub Pages settings and logs

Conclusion

Overall, creating a GitHub page was a rewarding experience that pushed my development skills. I gained confidence in web design, version control, and static website hosting, all of which have been crucial in my development journey.
