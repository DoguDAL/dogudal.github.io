GitHub Page Documentation

Step 1: Initial Setup

The first step in creating a GitHub page is setting up your repository. Make sure that your repository is public and named in the format username.github.io. This is crucial as GitHub automatically maps this format to serve the site at https://username.github.io/. 


            git init
            git remote add origin https://github.com/username/username.github.io.git

Step 2: Designing the Webpage

My initial struggle was designing a clean, responsive webpage. I had limited knowledge of CSS at the time, but I managed to learn by looking at templates, and using tools like Bootstrap helped me get a basic layout running quickly.

Step 3: Publishing the Site

Publishing the site was straightforward, but I initially faced problems with ensuring the correct branch was set for GitHub Pages. I learned that for project pages, I needed to configure it to use the gh-pages branch: 


            git checkout -b gh-pages
            git push origin gh-pages

After pushing the branch, I checked the repository settings and ensured the source branch for GitHub Pages was set to gh-pages.

Step 4: Struggles Faced

One of the most significant challenges was version control with HTML, CSS, and JavaScript. I initially faced difficulty keeping track of changes across different branches. However, learning more about Git commands like git log, git stash, and git diff helped me a lot.
