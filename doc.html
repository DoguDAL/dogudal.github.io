GitHub Webpage Creation Documentation

Author: Doğu Dal

Date: September 2024

Overview

This documentation explains the step-by-step process I followed to create my personal GitHub webpage. It covers the challenges I faced, how I overcame them, and the key lessons learned throughout the project.

Step 1: Setting up GitHub Pages

To start, I created a new repository on GitHub named dogudal.github.io. This is a crucial step because GitHub uses this repository name format to publish the webpage.

I used the following commands to clone the repository to my local machine:

git clone https://github.com/DoguDAL/dogudal.github.io.git

Challenges and Solutions

The first challenge I faced was understanding how to clone the repository. Initially, I tried to navigate to a non-existent directory, but I quickly realized the mistake and corrected it by navigating to the correct directory.

What I Learned

I learned how important it is to be attentive to directory paths while working with Git and GitHub. Proper navigation helps avoid common errors.

Step 2: Creating HTML Files

Once the repository was cloned, I created the necessary HTML files for the webpage: index.html, projects.html, cs50.html, and swift.html.

Here is an example of the command I used to create the index.html file:

touch index.html

Challenges and Solutions

I wasn’t familiar with opening HTML files from the terminal initially, so I had to learn the appropriate commands for opening the files in a text editor:

open -e index.html

This allowed me to edit the file content easily.

What I Learned

I learned how to use terminal commands to create and open files in a text editor. This saved me a lot of time when creating multiple files for the project.

Sample Code for index.html

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Home</title>
</head>
<body>
    <header>
        <nav>
            <a href="index.html">Home</a>
            <a href="projects.html">Projects</a>
            <a href="cs50.html">CS 50 Course</a>
            <a href="swift.html">Develop in Swift</a>
        </nav>
    </header>
    <main>
        <h1>Welcome to My Website</h1>
        <p>This is the home page.</p>
    </main>
</body>
</html>

Step 3: Committing and Pushing Changes

After editing the HTML files, I committed and pushed the changes to GitHub using the following commands:

git add .
git commit -m "Added HTML files for website sections"
git push origin main

Challenges and Solutions

I encountered a Git configuration message, prompting me to set my username and email explicitly. I resolved this by running the following commands:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

What I Learned

This step taught me the importance of configuring Git properly for commit history and authentication. It also reinforced the importance of regularly committing changes to prevent losing progress.

Image

Step 4: Publishing the Website

Finally, the website was successfully published at https://dogudal.github.io. GitHub Pages automatically handled the hosting based on the repository name.

What I Learned

Through this project, I gained hands-on experience with Git, GitHub Pages, HTML, and file management via the terminal. I also learned how to resolve issues and document my progress effectively.

Conclusion

This project allowed me to deepen my understanding of web development and version control. By overcoming challenges and applying new skills, I successfully created and published a personal webpage using GitHub Pages.
